# xsgrok2 深度审核任务

请以两种角色分别深度审核此项目，然后综合出修改方案并执行代码修改。

## 角色1：中文产品经理
你是一位有10年经验的中文互联网产品经理，精通移动App产品设计。请从以下角度审核：

1. **用户流程完整性**：从打开App到完成第一部小说的完整流程是否顺畅？有没有断点或卡壳？
2. **功能缺失**：对标主流AI写作App（秘塔写作猫、彩云小梦等），我们缺了什么关键功能？
3. **信息架构**：导航结构是否合理？用户能快速找到想要的功能吗？
4. **交互设计**：操作是否直觉化？有没有需要反复点击的冗余操作？
5. **内容策略**：生成小说的prompt质量如何？能不能产出真正可读的内容？
6. **通用性**：是否真的适配所有类型？言情/悬疑/日常/搞笑和玄幻的需求差异很大
7. **留存与粘性**：用户为什么要回来？有没有持续使用动机？

## 角色2：中文深度用户
你是一位中文AI小说的重度用户，每月消费10万字以上的AI生成内容。请从以下角度审核：

1. **创作体验**：我想写一部都市言情，从创建到看到第一章，体验好吗？
2. **内容质量**：生成的内容真的能读吗？有没有"AI味"太重的问题？
3. **编辑能力**：我能修改生成的设定和内容吗？修改方便吗？
4. **阅读体验**：章节阅读界面舒适吗？字体、行距、夜间模式？
5. **连续性**：第5章还记得第1章的设定吗？上下文衔接好不好？
6. **控制力**：我能控制故事走向吗？比如"这章要有个大反转"
7. **实用性**：我真的会用这个App来写小说吗？还是只是玩一下就卸了？
8. **中文适配**：全中文界面和内容？还是中英混杂？

## 审核后执行
1. 分别列出两个角色的审核意见（每个至少10条具体问题）
2. 综合两个角色的意见，按优先级排序（P0必须修/P1应该修/P2可以修）
3. 直接修改代码实现P0和P1的修改
4. 推送到GitHub，确保构建通过

## 当前代码问题提示（已知问题）
- UI全是英文，需要全中文化
- prompt太通用，没有根据类型调整
- 没有章节内容编辑功能
- 没有夜间模式/阅读设置
- 没有小说导出功能
- 阅读界面太简陋
- extractSection解析太脆弱
- 没有流式输出（streaming），用户要等很久
- 没有写作风格选择
- 没有章节大纲调整功能
